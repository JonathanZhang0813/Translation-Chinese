## 这个模型是什么？

该模型显示了一个简单的化学系统如何根据初始反应物的浓度达到不同的平衡状态。平衡是我们用来描述没有宏观变化的系统的术语。这意味着系统“看起来”好像什么都没有发生。事实上，在所有化学系统中，原子水平的过程都在继续，但在宏观水平上不会产生任何变化的平衡。

该模型模拟了四个分子的两个简单反应。反应可以写成：

```文本
A + B =======&gt; C + D
```

和

```文本
C + D =======&gt; A + B
```

这也可以写成一个单一的、可逆的反应：

```文本
A + B &lt;=======&gt; C + D
```

当一氧化碳与二氧化氮反应生成二氧化碳和一氧化氮（或一氧化氮）时，就会发生这种反应的经典现实例子。逆反应（二氧化碳和一氧化氮反应形成一氧化碳和二氧化氮）也是可能的。虽然反应中的所有物质都是气体，但我们实际上可以看到这样的系统达到平衡，因为二氧化氮 (NO <sub>2</sub> ) 是一种可见的微红色气体。当二氧化氮 (NO <sub>2</sub> ) 与一氧化碳 (CO) 结合时，生成的产物一氧化氮 (NO) 和二氧化碳 (CO <sub>2</sub> ) 是无色的，导致系统失去一些微红色。最终，系统与存在的一些“反应物”和一些“产物”达到平衡状态。

虽然系统最终会产生多少每种“反应物”和“产物”取决于许多因素（例如，包括物质反应时释放多少能量或系统温度），但该模型侧重于反应物的浓度。

## 这个模型是怎么运行的

在模型中，蓝色和黄色分子可以相互反应，绿色和粉色分子也可以。在每个周期中，每个分子在世界随机移动，遇到其他分子。如果它遇到可以与之反应的分子（例如黄色分子遇到蓝色，或粉红色遇到绿色，就会发生反应。因为蓝色和黄色分子反应产生绿色和粉红色分子，绿色和粉红色分子反应产生蓝色和黄色分子，最终达到平衡状态。

为了防止分子连续两次立即反应，每个分子都有一个“就绪计时器”。反应后，此计时器设置为 2，并且在两个周期中，计时器递减回零，允许分子再次反应。

## 如何这个模型怎么用

YELLOW-MOLECULES 和 BLUE-MOLECULES 滑块分别确定黄色和蓝色分子的起始数量。一旦设置了这些滑块，按下 SETUP 按钮就会创建这些分子并将它们分布到世界。

GO 按钮将模拟设置为运动。分子随机移动并相互反应，改变颜色以表示原子重排成不同的分子结构。系统很快达到平衡。

四个监视器显示系统中存在的每种分子的数量。图表MOLECULE AMOUNTS 显示了随着时间的推移存在的每种分子的数量。

## 看一看

您可能会注意到产物分子的数量受限于最少量的初始反应物分子。请注意，产品分子的数量总是相同的，因为它们是一一对应的。

## 试一试

两种反应物的不同量如何影响最终平衡？绝对数量重要吗，是数量之间的差异，还是重要的两种反应物的比率？

尝试在五个连续的模拟中将 YELLOW-MOLECULES滑块设置为 400，将 BLUE-MOLECULES滑块设置为 20、40、100、200 和 400。你在每种情况下预测什么样的平衡状态？某些比率是可预测的吗？

当您从相同数量的黄色分子和蓝色分子开始时会发生什么？运行模型后，这两类分子的计数有什么关系？

## 改一改

如果正向和反向反应速率由变量而不是初始浓度决定怎么办？您可以将这种模拟与该模型中的模拟进行比较，看看浓度和反应速率是否相互独立，如最终平衡状态所测量的那样。

您还可以通过允许用户在模拟运行时将新分子引入模拟来扩展程序。添加 50 个蓝色分子将如何影响已经处于平衡状态的系统？
