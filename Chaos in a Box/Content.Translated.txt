## 这个模型是什么？

该模型演示了一个非常简单的混沌系统，该系统由一个在带有圆形障碍物的矩形房间内弹跳的球组成。该系统可以表现出周期性行为（其中球以重复模式反弹），但主要表现出混沌行为。

混乱行为意味着初始条件（在本例中为球的位置和方向）的任何微小变化都会随着时间的推移导致截然不同的行为。混沌行为使系统很难精确预测：为了准确预测混沌系统，必须对系统的初始条件进行精确测量。然而，一般来说，以这种精确度测量一个系统是不可能的。

## 这个模型是怎么运行的

球从墙壁和障碍物反弹。碰撞都是完全弹性的，障碍物保持静止。最后，没有摩擦。这意味着球永远不会改变速度，只会改变方向。请注意，球不会相互反弹；多个球只是为了显示初始位置差异很小时会发生什么。

当球与障碍物或世界边缘重叠时，就会检测到碰撞。然后纠正该位置（有关更详细的讨论，请参见下面的 NetLogo 功能部分）。最后，因为碰撞是完全有弹性的，所以球的新角度是它的方向在它所碰撞的表面周围的反射角。

## 如何这个模型怎么用

使用 SETUP-RANDOM 时，使用 NUM-OBSTACLES滑块调整障碍物的数量。有 0 个障碍物，系统将*总是*是周期性的。你能弄清楚为什么吗？

如果两个球？打开时，设置按钮将创建一个额外的球，该球的位置几乎（但不完全）与第一个球相同。用两个球运行系统将显示初始条件的微小变化对混沌系统的影响有多大。

使用 SETUP-RANDOM 按钮以最可能混乱的配置初始化系统。 NUM-OBSTACLES 个随机大小的障碍物将被放置在随机位置。

使用 SETUP-PERIODIC-X 按钮查看系统可以展示的周期性行为的示例。

使用 SETUP-PERIODIC-QUILT 按钮查看另一个具有很长周期的周期性行为示例。

GO 按钮运行模拟。

如果是TRACE-PATH？开关打开，球的路径将被追踪。 CLEAR-DRAWING 清除路径。

DRAG 可用于移动球和障碍物，以及改变障碍物的大小和球的方向。按下后，在球和障碍物的中心附近单击并拖动会使它们四处移动。在球的边缘附近单击并拖动将改变它们的方向。在障碍物的边缘附近单击并拖动会改变它们的大小。

## 看一看

请注意，在球击中圆圈之前，球的弹跳是如何相当可预测的。你认为这是为什么？

如果您在没有障碍的情况下运行模型，则行为始终是周期性的，尽管该周期可能很长。为什么会这样？

如果两个球？打开时，设置按钮将创建一个额外的球，该球的位置几乎（但不完全）与第一个球相同。用两个球和一个障碍物运行系统将显示初始条件的微小变化对混沌系统的影响有多大。这是混沌系统的标志性特征。

球与障碍物碰撞的随机配置不太可能是周期性的。

## NetLogo 语言特性

该模型使用了许多先进的技术：

首先，匿名过程用于使 SETUP过程在球和障碍物的放置方面非常灵活。例如，SETUP-PERIODIC-QUILT 使用以下代码：

```
设置 [-&gt; setxy 0 8 设置大小 1 ] [-&gt; setxy 0 7 facexy -8 -6 ]
```

第一个匿名过程配置障碍物，第二个配置球。

FOREACH-AGENT过程展示了如何使用匿名过程在 NetLogo 中创建自己的控制结构。 FOREACH-AGENT 像主体集合一样工作，但在代理集而不是列表上运行。

其次，由于该系统的混沌特性，必须非常精确地处理对象之间的碰撞。因为球在每个周期上以 SPEED 大小的步长移动，所以当球击中另一个物体时，球所处的位置不会与球一直在连续移动一样。例如，此错误可能会使应定期配置的配置变得混乱。为了纠正这个问题，该模型使用了一种称为 [二进制搜索](https://en.wikipedia.org/wiki/Binary_search_algorithm) 的技术来非常快速地非常接近碰撞的确切位置。虽然二进制搜索是一种通用算法，而不是 NetLogo 功能，但该模型展示了如何在 NetLogo 中实现空间二进制搜索。此外，由于 ATAN报告指令器可以引入小的不精确性，该模型在与障碍物碰撞后将球的方向圆整到最接近的 100,000 度。

最后，拖动障碍物和球的代码展示了如何用鼠标完成相当复杂的主体操作。
